# ブレインダンプアプリ 機能仕様書

## 1. 文書の目的

本書は、ブレインダンプアプリが**何をするか**と**どのように動くか**を定義する。  
実装者が `index.html` 内の JavaScript と Supabase SDK で CRUD を実装できる粒度を目指す。

## 3. システム前提

| 項目 | 内容 |
|---|---|
| フロントエンド | HTML / CSS / JavaScript（バニラ） |
| ファイル構成 | まずは `index.html` 1ファイル |
| パッケージ管理 | npm は使わない |
| Supabase SDK | CDN の UMD 版から読み込む（`@supabase/supabase-js`） |
| 対象テーブル | `public.brain_dumps`（メモ）、`public.categories`（カテゴリマスタ） |
| 認証 | 今回は行わない（anon key でアクセス） |
| RLS | 今回は使用しない（DB側で無効、またはポリシー未設定想定） |
| ユーザー識別 | `user_id` は使わない |

### 3.1 今回の機能で使うテーブル列

**`brain_dumps`（メモ本体）**

| カラム名 | 新規保存 | 一覧 | 編集 | 削除 |
|---|---|---|---|---|
| id | DBが自動生成（返却値を保持） | 内部利用 | 更新・削除のキー | 削除のキー |
| title | 送信（必須） | 表示 | 送信 | - |
| category_id | 送信（任意、`null` 可。`categories.id` を参照） | カテゴリ名解決に利用 | 送信 | - |
| content | 送信（必須） | 表示 | 送信 | - |
| next_action | 送信（任意、`null` 可） | 表示 | 送信 | - |
| created_at | 送らない（DBデフォルト） | 表示 | 送らない | - |

**`categories`（カテゴリマスタ）**

| カラム名 | 機能での扱い |
|---|---|
| id | ドロップダウンの `value`。`brain_dumps.category_id` の参照先 |
| name | ドロップダウン文言・一覧のカテゴリ列の表示 |
| sort_order | ドロップダウンの並び順（昇順） |

DBの詳細は `docs/02_DB仕様書.md` を参照すること。

### 3.2 Supabase操作の用語（補足）

| 用語 | 意味 |
|---|---|
| insert | 新しい行（メモ）をテーブルに追加する |
| select | テーブルからデータを読み取る |
| update | 既存の行の内容を書き換える |
| delete | 既存の行を削除する |

## 5. 機能詳細

### 5.1 メモ新規保存機能

#### 目的

ユーザーが入力した各項目を、`brain_dumps` テーブルに新しい1件として保存する。

#### 入力値

| 項目 | 必須 | 入力方式 | 送信先カラム |
|---|---|---|---|
| タイトル | はい | テキスト | `title` |
| カテゴリ | いいえ | ドロップダウン選択（`categories` から） | `category_id`（uuid） |
| 本文 | はい | テキスト | `content` |
| 次のアクション | いいえ | テキスト | `next_action` |

送信しない項目：`id`（自動）、`created_at`（自動）

任意項目の扱い：
- カテゴリは未選択（空 `value`）のときは `null` を送る。
- `next_action` は前後の空白を除いて空のときは `null` を送る（空文字は送らない）。

#### 処理の流れ

1. ユーザーが各入力欄に入力する
2. 「保存」ボタンを押す
3. 画面側で入力チェック（タイトル・本文は空欄・空白のみ不可）
4. チェックOKなら Supabase へ **insert** を実行
5. 成功したら：
   - 成功メッセージを表示
   - すべての入力欄を空にする
   - 編集中ID（保持していれば）をクリア
   - メモ一覧を再読み込み（select）
6. 失敗したらエラーメッセージを表示し、入力内容は残す

#### 出力・表示内容

- 一覧に新しいメモが先頭付近（新しい順）に表示される
- メッセージ例：「保存しました」

#### 正常時の動き

- DBに1行追加される
- 一覧が最新状態になる

#### エラー時の動き

| ケース | 表示例 |
|---|---|
| タイトル未入力 | 「タイトルを入力してください」 |
| 本文未入力 | 「本文を入力してください」 |
| Supabaseエラー | 「保存に失敗しました。しばらくしてから再度お試しください」 |

#### Supabase対応

- 操作：**insert**
- テーブル：`brain_dumps`
- 送信データ例：`{ title: "...", category_id: "9f...uuid", content: "...", next_action: null }`
- カテゴリ未選択時の例：`{ title: "...", category_id: null, content: "...", next_action: null }`

### 5.2 メモ一覧表示機能

#### 目的

保存済みのメモを画面に一覧表示し、編集・削除の対象を選べるようにする。

#### 入力値

なし（画面表示時・他機能完了後に自動実行）

#### 処理の流れ

1. ページ読み込み完了後、Supabase へ **select** を実行
2. `created_at` の**新しい順**（降順）で並べる
3. 取得結果を配列 `allMemos` に保持する
4. 機能 7（カテゴリ絞り込み）の条件でフィルタし、メモ一覧エリアに描画する
5. 各メモに「編集」「削除」ボタンを付ける
6. 0件のときは案内文を表示する（DB上0件と、絞り込み結果0件で文言を分ける）

#### 出力・表示内容

| 表示項目 | データ元 |
|---|---|
| タイトル | `title` |
| 作成日時 | `created_at`（表示用に整形） |
| カテゴリ名 | `categories.name`（`category_id` を介して結合 or マッピング解決。未設定時は「—」など） |
| 次のアクション | `next_action`（未設定時は「—」など） |
| 本文 | `content` 全文または抜粋 |
| 編集ボタン | - |
| 削除ボタン | - |

#### 正常時の動き

- 一覧にメモが表示される
- 最新のメモが上に来る

#### エラー時の動き

| ケース | 表示例 |
|---|---|
| 取得失敗 | 「一覧の読み込みに失敗しました」 |
| 読み込み中 | 「読み込み中…」（任意） |

#### Supabase対応

- 操作：**select**
- テーブル：`brain_dumps`
- 並び順：`created_at` 降順
- 取得列：`id`, `title`, `category_id`, `content`, `next_action`, `created_at`, および結合で `categories ( name )`
- カテゴリ名は `categories` を結合して取得する。結合できない場合は機能 6 で取得済みのカテゴリマップ（`id → name`）で解決してもよい

※ 保存・更新・削除後の一覧再取得でも、ユーザーが選んでいる絞り込み条件（`filterCategoryId`）は維持する。

### 5.3 メモ編集機能

#### 目的

既存メモの各項目を書き換えて保存する。

#### 入力値

| 項目 | 必須 | 説明 |
|---|---|---|
| タイトル | はい | テキスト入力欄から取得 |
| カテゴリ | いいえ | ドロップダウンから取得。未選択（空 value）なら `null` |
| 本文 | はい | テキスト入力欄から取得 |
| 次のアクション | いいえ | テキスト入力欄から取得。空なら `null` |
| 編集中の id | はい（内部） | 編集ボタン押下時に保持。画面には表示しなくてよい |

#### 処理の流れ

1. 一覧の「編集」ボタンを押す
2. 対象メモの `id`, `title`, `category_id`, `content`, `next_action` を取得する
3. `id` を JavaScript 変数（例：`editingId`）に保存する
4. 各項目を対応する入力欄・ドロップダウンにセットする  
   （カテゴリは `select.value = data.category_id ?? ""`）
5. 保存ボタンの文言を「更新」に変えてもよい
6. ユーザーが内容を直し、「保存」/「更新」を押す
7. 入力チェック後、**update** を実行（条件：`id` が一致）
8. 成功したら：
   - 成功メッセージを表示
   - 入力欄を空にする
   - `editingId` をクリア（新規モードに戻す）
   - 一覧を再読み込み
9. 失敗したらエラーメッセージを表示

#### 出力・表示内容

- メッセージ例：「更新しました」
- 一覧上の該当メモの各項目が新しい内容になる

#### 正常時の動き

- 該当行のみ `title`, `category_id`, `content`, `next_action` が更新される
- `created_at` は変更しない

#### エラー時の動き

| ケース | 表示例 |
|---|---|
| 編集対象未選択のまま更新 | 「編集するメモを選んでください」（通常は発生しにくい） |
| 入力不備 | 新規保存と同様のバリデーションメッセージ |
| update失敗 | 「更新に失敗しました」 |

#### Supabase対応

- 操作：**update**
- テーブル：`brain_dumps`
- 条件：`id` が編集中の id と一致
- 更新データ：`{ title, category_id, content, next_action }`

---

### 5.4 メモ削除機能

#### 目的

不要になったメモをテーブルから削除する。

#### 入力値

| 項目 | 必須 | 説明 |
|---|---|---|
| 削除対象の id | はい | 削除ボタンに紐づくメモの `id` |

#### 処理の流れ

1. 一覧の「削除」ボタンを押す
2. ブラウザの **confirm** で確認ダイアログを表示  
   例：「このメモを削除しますか？」
3. ユーザーがキャンセル → 何もしない
4. OK → Supabase へ **delete** を実行（条件：`id` 一致）
5. 成功したら：
   - 成功メッセージを表示
   - 削除対象が編集中だった場合は入力欄と `editingId` をクリア
   - 一覧を再読み込み
6. 失敗したらエラーメッセージを表示

#### 出力・表示内容

- メッセージ例：「削除しました」
- 一覧から該当メモが消える

#### 正常時の動き

- DBから該当行が削除される
- 一覧が最新状態になる

#### エラー時の動き

| ケース | 表示例 |
|---|---|
| ユーザーがキャンセル | メッセージなし |
| delete失敗 | 「削除に失敗しました」 |

#### Supabase対応

- 操作：**delete**
- テーブル：`brain_dumps`
- 条件：`id` が削除対象の id と一致

### 5.5 メッセージ表示機能

#### 目的

ユーザーに操作結果やエラー内容を短く伝え、次に何をすればよいか分かるようにする。

#### 入力値

JavaScript 内でメッセージ種別と文言を決定する。

#### 処理の流れ

1. 各機能（保存・更新・削除・一覧取得）の成功/失敗時に呼び出す
2. メッセージ表示エリアに文言をセットする
3. 必要に応じて CSS クラスで成功/エラーの見た目を切り替える
4. 数秒後に消す、または次のメッセージで上書きする

#### メッセージ種別一覧

| 種別 | 表示例 | 見た目 |
|---|---|---|
| 保存成功 | 保存しました | 成功スタイル |
| 更新成功 | 更新しました | 成功スタイル |
| 削除成功 | 削除しました | 成功スタイル |
| 入力エラー | タイトルを入力してください など | エラースタイル |
| Supabase接続エラー | 通信に失敗しました。接続を確認してください | エラースタイル |
| その他エラー | 処理に失敗しました | エラースタイル |

#### 正常時の動き

- ユーザーが結果を認識できる

#### エラー時の動き

- 技術的な詳細（スタックトレース等）は画面に出さず、コンソールログに留めてもよい

#### Supabase対応

- なし（画面のみの機能）

### 5.6 カテゴリ選択肢取得機能

#### 目的

メモの投稿・編集フォームのカテゴリドロップダウン、および一覧の絞り込み用ドロップダウンに表示する選択肢を、`categories` テーブルから取得する。

#### 入力値

なし（画面表示時に自動実行）。

#### 処理の流れ

1. ページ読み込み時に Supabase へ **select** を実行する
2. `sort_order` 昇順で取得する
3. 取得結果をもとに**入力フォーム用**ドロップダウンを組み立てる  
   - 先頭に「未選択」（空 `value`）を必ず置く  
   - 続いて各カテゴリを `<option value="categories.id">categories.name</option>` で追加する
4. 同じ取得結果をもとに**絞り込み用**ドロップダウンを組み立てる  
   - `すべて`（`value="all"`）→ `未分類`（`value="none"`）→ 各カテゴリ（`value=categories.id`）の順  
   - 構築後、現在の `filterCategoryId` を `select.value` に復元する
5. `id → name` のマッピングを内部変数（例：`categoriesById`）に保持しておく（一覧表示で結合できなかった場合のフォールバック用）

#### 出力・表示内容

- フォーム上のカテゴリドロップダウンに選択肢が並ぶ
- 一覧側で `category_id` からカテゴリ名を解決できる

#### 正常時の動き

- ドロップダウンが利用可能になる

#### エラー時の動き

| ケース | 表示例 |
|---|---|
| 取得失敗 | 「カテゴリの読み込みに失敗しました」（ドロップダウンは「未選択」のみのまま） |

#### Supabase対応

- 操作：**select**
- テーブル：`categories`
- 並び順：`sort_order` 昇順
- 取得列：`id`, `name`

### 5.7 カテゴリ絞り込み機能

#### 目的

保存済みメモの一覧を、ユーザーが選んだカテゴリ条件だけに絞って表示する。

#### 入力値

| 項目 | 説明 |
|---|---|
| 絞り込み条件 | 絞り込み用 `<select>` の `value`。`all` / `none` / カテゴリの uuid |

#### 処理の流れ

1. 機能 5.2 で取得した全メモを `allMemos` に保持している前提とする
2. ユーザーが絞り込み用 `<select>` を変更する
3. 選択値を `filterCategoryId` に保存する
4. `applyCategoryFilter()` で `allMemos` をフィルタする  
   - `all`：全件  
   - `none`：`category_id` が `null` のみ  
   - それ以外：`category_id` が選択 uuid と一致するもののみ
5. フィルタ結果を `renderMemoList()` で一覧に描画する（DBへの再 select は行わない）

#### 出力・表示内容

- 条件に合うメモのみ一覧に表示される
- 合致0件かつ `allMemos` に1件以上あるとき：「該当するメモはありません」
- `allMemos` が0件のとき：「メモはまだありません」

#### 正常時の動き

- 選択を変えるたびに一覧が即時更新される
- 新規保存・更新・削除後に一覧を再取得しても、直前の `filterCategoryId` は維持される

#### エラー時の動き

- 本機能単体では DB を呼ばないため、専用のエラー表示は不要（一覧取得失敗は機能 5.2 に従う）

#### Supabase対応

- なし（クライアント側フィルタのみ）

## 6. 画面読み込み時の処理順（推奨）

実装時の初期化の目安。

1. Supabase クライアントを初期化（URL・anon key は後で設定）
2. カテゴリ選択肢取得（`categories` select）を実行して入力用・絞り込み用ドロップダウンを構築
3. メモ一覧表示（`brain_dumps` select）を実行し、`allMemos` に保持したうえで絞り込みを適用
4. ボタン・フォーム・絞り込み `<select>` のイベントを登録

## 7. 内部で保持する変数（推奨）

| 変数名（例） | 用途 |
|---|---|
| `editingId` | 編集中メモの `id`。`null` のときは新規保存モード |
| `supabaseClient` | Supabase クライアントインスタンス |
| `categoriesById` | カテゴリ `id → name` のマッピング。一覧表示でのカテゴリ名解決に利用 |
| `allMemos` | 一覧取得の全件。絞り込みの元データ |
| `filterCategoryId` | 現在の絞り込み条件（`all` / `none` / uuid） |

## 8. バリデーション共通ルール

| ルール | 内容 |
|---|---|
| タイトル | 前後の空白を除いて空でないこと |
| 本文 | 前後の空白を除いて空でないこと |
| カテゴリ | 任意。ドロップダウン未選択のときは `category_id` に `null` を保存 |
| 次のアクション | 任意。空白のみのときは `null` として保存 |

## 9. 今回作らない機能

| 対象外 |
|---|

| 画像アップロード |
| ページ遷移（複数HTML） |
| npm ビルド・React / Next.js |
| 外部API連携 |
| 本番デプロイ手順 |
| `.mdc` ルールファイル |

## 10. 実装時の注意（セキュリティ）
