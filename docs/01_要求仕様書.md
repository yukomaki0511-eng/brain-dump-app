# ブレインダンプアプリ 要求仕様書

## 1. 目的
- 思考・アイデアを素早く記録し、後から振り返れる自分専用メモアプリを提供する。
- Supabase を利用して認証とデータ保存を行い、スマホ/PCの両方から同一データにアクセスできるようにする。

## 2. 対象ユーザー
- アプリ利用者は本人のみ（シングルユーザー想定）。
- ただし実装上はログインユーザーごとにデータ分離される設計とする。

## 3. 前提技術
- BaaS: Supabase
- 認証: Supabase Auth
- データベース: Supabase PostgreSQL
- データ構造: メモテーブル `brain_dumps`、カテゴリマスタテーブル（専用テーブル）

## 4. 機能要件
### 4.1 メモ作成
- ユーザーはタイトルと本文を入力してメモを保存できる。
- 保存時に作成日時を自動で記録する。
- タイトルと本文は必須とする。

### 4.2 メモ一覧表示
- ログイン中ユーザーは保存済みメモの一覧を閲覧できる。
- 一覧には最低限、タイトルと作成日時を表示する。
- 作成日時の新しい順で表示する。
- カテゴリで一覧を絞り込める（すべて／未分類／各カテゴリ）。絞り込みは画面側で行い、選択中の条件は保存・削除・更新後も維持する。
- 一覧の本文・次のアクションに `http://` または `https://` で始まる URL が含まれる場合、クリック可能なリンクとして表示する（新しいタブで開く）。それ以外の文字列はプレーンテキストのまま表示する。

### 4.3 メモ編集
- ユーザーは既存メモのタイトル・本文を更新できる。
- 更新結果は一覧および詳細表示に反映される。

### 4.4 メモ削除
- ユーザーは既存メモを削除できる。
- 削除後は一覧から対象メモが表示されなくなる。

### 4.5 ログインとデータ分離
- ユーザーはログイン後にのみメモ機能を利用できる。
- 自分のデータのみ参照・更新・削除できる。
- 他ユーザーのデータにはアクセスできない（RLSで制御）。

### 4.6 マルチデバイス利用
- スマホ・PCのどちらからでも同一アカウントで利用できる。
- レスポンシブUIで主要画面（ログイン、一覧、作成/編集）を操作可能にする。

### 4.7 カテゴリ管理機能
- メモにカテゴリを設定できる。
- カテゴリの選択肢は、専用のマスタテーブルで管理する。
- メモの投稿・編集時に、ドロップダウンでカテゴリを選べる。
- カテゴリの入力は必須項目にしない（未選択可）。
- 一覧画面にカテゴリ名を表示する。

## 5. 非機能要件
- 可用性: Supabase の標準可用性に準拠。
- セキュリティ: 認証必須、RLS有効化、SQLインジェクション対策（SDK経由アクセス）を行う。
- 操作性: 入力と保存の導線をシンプルにし、最小ステップでメモ登録できること。
- 保守性: テーブル構成はシンプルな1テーブルで開始し、将来拡張を妨げない設計にする。

## 6. 画面要件（最小）
- ログイン画面
- メモ一覧画面
- メモ作成/編集画面

## 7. スコープ
### 7.1 本仕様に含む
- CRUD（作成・一覧・編集・削除）
- カテゴリ管理（マスタテーブル、ドロップダウン選択、一覧表示、一覧のカテゴリ絞り込み）
- 一覧表示時の URL 自動リンク化（本文・次のアクション）
- 認証
- ユーザー単位のデータアクセス制御
- スマホ/PC対応

### 7.2 本仕様に含まない（初期リリース対象外）
- 添付ファイル機能
- タグ機能
- 全文検索・複合条件フィルタなど、検索の高度化
- 共有機能
