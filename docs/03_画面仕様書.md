# ブレインダンプアプリ 画面仕様書

## 1. 文書の目的

本書は、ブレインダンプアプリの**画面構成**と**画面上の要素**を定義する。  
実装者が `index.html` 1ファイルで画面を組み立てられる粒度を目指す。

## 3. アプリ概要

| 項目 | 内容 |
|---|---|
| アプリ名 | ブレインダンプ |
| 目的 | 思考・アイデアをテキストで入力して保存し、あとから一覧で見返す |
| 対象 | 自分専用のメモアプリ |
| 画面数 | 1画面（`index.html` にすべて配置） |
| 技術 | HTML / CSS / JavaScript のみ（フレームワーク不使用） |
| データ保存 | Supabase の `brain_dumps` テーブル（カテゴリは `categories` マスタを参照） |

### 3.1 今回の画面で使うDBカラム

画面から操作・表示するカラムは次のとおり。

**`brain_dumps`**

| カラム名 | 画面での扱い |
|---|---|
| id | 編集・削除時に内部で保持（画面には一覧で間接的に利用） |
| title | 入力欄・一覧表示（必須） |
| category_id | ドロップダウン選択・一覧表示（任意、`null` 可） |
| content | 入力欄・一覧表示（必須） |
| next_action | 入力欄・一覧表示（任意） |
| created_at | 一覧表示のみ（DBが自動設定） |

**`categories`**（カテゴリマスタ。ドロップダウンの選択肢用に参照）

| カラム名 | 画面での扱い |
|---|---|
| id | ドロップダウンの `value`（`brain_dumps.category_id` に保存される） |
| name | ドロップダウンの表示文言・一覧のカテゴリ列に表示 |
| sort_order | ドロップダウンの並び順（昇順） |

詳細は `docs/02_DB仕様書.md` を参照すること。

## 5. メイン画面

### 5.1 画面名

**メイン画面**

### 5.2 画面の目的

- 新しいメモ（タイトル・カテゴリ・本文・次のアクション）を入力して保存できる
- 保存済みメモを一覧で見られる
- カテゴリで一覧を絞り込める
- 既存メモを編集・削除できる
- 操作結果（成功・失敗）を画面上のメッセージで伝える

### 5.3 画面レイアウト（イメージ）

画面上から次の順で並べる。

1. アプリタイトル
2. メッセージ表示エリア
3. 入力フォーム（タイトル・カテゴリ・本文・次のアクション・保存ボタン）
4. メモ一覧エリア（カテゴリ絞り込み・メモリスト）

PCでは縦に読みやすい幅（例：最大幅 720px 前後）で中央寄せ。  
スマホでは画面幅いっぱいに使い、タップしやすいサイズにする。

## 6. 画面に必要な要素

### 6.1 要素一覧と役割

| 要素名 | HTML要素の例 | 役割 |
|---|---|---|
| アプリタイトル | `h1` | アプリ名「ブレインダンプ」を表示し、何のアプリか一目で分かるようにする |
| メッセージ表示エリア | `div`（id 付き） | 保存成功・エラーなどの短いメッセージを表示する。通常は空 |
| タイトル入力欄 | `input`（type="text"） | メモのタイトルを入力する。必須 |
| カテゴリ選択欄 | `select`（ドロップダウン） | `categories` テーブルから取得した選択肢を表示する。任意（未選択可） |
| 本文入力欄 | `textarea` | メモの本文を入力する。必須。複数行入力できること |
| 次のアクション入力欄 | `input`（type="text"） | 次に取る行動を入力する。任意 |
| 保存ボタン | `button` | 新規保存または編集内容の更新を実行する |
| 編集をやめるボタン | `button` | 編集モードを解除し、入力欄を空にする（編集中のみ表示） |
| カテゴリ絞り込み欄 | `select`（id: `filter-category-select`） | 一覧をカテゴリで絞り込む。選択肢は「すべて」「未分類」およびマスタの各カテゴリ |
| メモ一覧エリア | `section` + リスト | 保存済みメモをカードまたは行形式で並べて表示する（絞り込み結果のみ） |
| 編集ボタン | 各メモ内の `button` | そのメモの内容を入力欄に読み込み、編集モードにする |
| 削除ボタン | 各メモ内の `button` | そのメモを削除する（確認後） |

### 6.2 各要素の詳細

#### アプリタイトル

- 表示文言：`ブレインダンプ`（または短いサブタイトル付きでも可）
- ページの最上部に固定表示

#### メッセージ表示エリア

- 成功時：緑系など、見た目で「成功」と分かるスタイル
- エラー時：赤系など、見た目で「注意」と分かるスタイル
- 数秒後に自動で消える、または次の操作で上書きする

#### タイトル入力欄

- ラベル例：「タイトル」
- プレースホルダ例：「例：今日の気づき」
- 1行入力

#### カテゴリ選択欄（ドロップダウン）

- ラベル例：「カテゴリ」
- 要素：`select`
- 先頭に「未選択」（空 value）を必ず置く
- 残りの選択肢は `categories` テーブルから取得し、`sort_order` 昇順で並べる
- 各選択肢の `value` は `categories.id`（uuid）、表示文言は `categories.name`
- 未選択可。未選択のときは `brain_dumps.category_id` に `null` を保存
- ページ読み込み時に1度だけ取得する（編集モード切替時も再取得しない）

#### 本文入力欄

- ラベル例：「本文」
- プレースホルダ例：「思いついたことを書いてください」
- 高さはスマホでも十分に書ける行数（例：6行以上）

#### 次のアクション入力欄

- ラベル例：「次のアクション」
- プレースホルダ例：「例：明日メモを整理する」
- 1行入力
- 未入力可（空のときは DB に `null` を保存）

#### 保存ボタン

- 新規時の表示例：「保存」
- 編集中の表示例：「更新」（同じボタンで切り替えてもよい）
- 処理中は二重送信を防ぐため、一時的に無効化してもよい

#### カテゴリ絞り込み欄

- ラベル文言：`カテゴリで絞り込み`
- 配置：メモ一覧エリア（「保存済みメモ」見出し）の直下、リストの直上
- 選択肢（`value` → 表示文言）：
  - `all` → `すべて`
  - `none` → `未分類`（`category_id` が `null` のメモのみ）
  - 各 `categories.id` → `categories.name`（`sort_order` 昇順）
- 変更時：一覧を再描画する（DB再取得はしない）
- カテゴリマスタ読み込み後に選択肢を構築する（入力フォーム用ドロップダウンと同じマスタを利用）

#### メモ一覧エリア

- DB上にメモが0件のとき：「メモはまだありません」などの案内文を表示
- メモはあるが絞り込み結果が0件のとき：「該当するメモはありません」などの案内文を表示
- 1件以上表示対象があるとき：各メモをブロックで表示

#### 各メモブロック内の表示

| 表示項目 | 説明 |
|---|---|
| タイトル | メモの `title` |
| 作成日時 | `created_at` を人が読みやすい形式で表示（例：2026/05/10 21:30） |
| カテゴリ名 | `brain_dumps.category_id` に紐づく `categories.name`（結合 or マッピングで解決）。未設定のときは「—」などで表示 |
| 次のアクション | メモの `next_action`。未設定のときは「—」などで表示 |
| 本文の抜粋 | 長い場合は先頭数十文字＋「…」など（全文表示でも可） |
| 編集ボタン | そのメモを編集対象にする |
| 削除ボタン | そのメモを削除する |

---

## 7. ボタン一覧

| ボタン名 | 表示位置 | 押したときの動き |
|---|---|---|
| 保存 / 更新 | 入力フォーム下 | 新規なら insert、編集中なら update を実行 |
| 編集 | 各メモ行・カード内 | 選択メモの title / category_id / content / next_action を入力欄・ドロップダウンにセットし、編集モードへ |
| 編集をやめる | 入力フォーム内（編集中のみ） | 編集モードを解除し、入力欄をクリアする |
| 削除 | 各メモ行・カード内 | 確認ダイアログ後、delete を実行 |

---

## 8. 入力項目一覧

| 項目名 | 必須 | 入力方式 | バリデーション（画面側） |
|---|---|---|---|
| タイトル | 必須 | テキスト入力（実用上 200 文字程度） | 空欄・空白のみは不可 |
| カテゴリ | 任意 | ドロップダウン選択（`categories` から取得） | 未選択可。未選択のときは `category_id` に `null` を保存 |
| 本文 | 必須 | 複数行テキスト入力（実用上 5000 文字程度） | 空欄・空白のみは不可 |
| 次のアクション | 任意 | テキスト入力（実用上 200 文字程度） | 未入力可。空白のみのときは `null` として保存 |

※ `title` / `content` / `next_action` の DB 型は `text` のため、文字数上限は DB 側では設けず、必須項目のみ空チェックを画面側で行う。  
※ カテゴリは自由入力ではなく、必ずマスタの選択肢から選ばせる（自由入力で新規カテゴリを増やす操作は今回は作らない）。

## 9. 表示項目一覧

| 表示項目 | データの出どころ | 表示場所 |
|---|---|---|
| アプリタイトル | 固定文言 | ページ上部 |
| 操作メッセージ | JavaScript で生成 | メッセージ表示エリア |
| タイトル（入力中） | ユーザー入力 | タイトル入力欄 |
| カテゴリ（選択中） | ユーザー選択 | カテゴリ選択欄（ドロップダウン） |
| カテゴリの選択肢 | `categories.id` / `categories.name`（`sort_order` 昇順） | カテゴリ選択欄・カテゴリ絞り込み欄の `<option>` |
| 絞り込み条件（選択中） | ユーザー選択（`all` / `none` / uuid） | カテゴリ絞り込み欄 |
| 本文（入力中） | ユーザー入力 | 本文入力欄 |
| 次のアクション（入力中） | ユーザー入力 | 次のアクション入力欄 |
| メモのタイトル | `brain_dumps.title` | 一覧の各メモ |
| メモの作成日時 | `brain_dumps.created_at` | 一覧の各メモ |
| メモのカテゴリ名 | `categories.name`（`brain_dumps.category_id` を介して取得） | 一覧の各メモ |
| メモの次のアクション | `brain_dumps.next_action` | 一覧の各メモ |
| メモの本文（全文または抜粋） | `brain_dumps.content` | 一覧の各メモ |

## 10. 画面の状態

画面は次の状態を取る。実装時は JavaScript の変数（例：`editingId`）で編集モードを管理する。

### 10.1 状態一覧

| 状態名 | 説明 | 画面上の見え方 |
|---|---|---|
| 初期表示 | ページを開いた直後 | カテゴリ選択肢の読み込み後、入力欄は空。一覧を読み込み中、または一覧表示 |
| 入力中 | ユーザーが各入力欄に入力している | 特別な表示変化はなく、通常の入力フォーム |
| 保存成功 | 新規 insert が成功 | 成功メッセージ、入力欄クリア、一覧更新 |
| 保存失敗 | 新規 insert が失敗 | エラーメッセージ、入力内容は残す |
| 一覧表示 | 一覧の取得が完了 | メモ一覧にデータが表示される（0件含む） |
| 編集中 | いずれかのメモを編集対象にしている | 入力欄に選択メモの全項目、保存ボタンは「更新」、「編集をやめる」表示 |
| 更新成功 | update が成功 | 成功メッセージ、入力欄クリア、編集モード解除、一覧更新 |
| 削除確認 | 削除ボタン押下後 | ブラウザの `confirm` ダイアログを表示 |
| 削除成功 | delete が成功 | 成功メッセージ、一覧更新 |
| エラー発生時 | 通信エラー・DBエラーなど | エラーメッセージを表示 |

### 10.2 状態遷移（簡易）

```
初期表示 → 一覧表示
入力中 → 保存成功 → 一覧表示（入力欄空）
入力中 → 保存失敗 → 入力中（内容保持）
一覧表示 → 編集中 → 更新成功 → 一覧表示（入力欄空）
一覧表示 → 削除確認 → 削除成功 → 一覧表示
いずれの状態でも → エラー発生時（メッセージ表示）
```

## 11. スマホ表示で注意すること

| 項目 | 注意内容 |
|---|---|
| タップ領域 | ボタンは指で押しやすい高さ（目安 44px 以上）にする |
| 入力欄 | `font-size` を 16px 以上にし、iOS で入力時に画面が不必要に拡大しないようにする |
| 本文欄 | 縦方向に十分な高さを確保する |
| 一覧 | 1列レイアウト。横スクロールが発生しないようにする |
| 余白 | 画面端と要素の間に十分な `padding` を取る |
| 削除確認 | スマホでも `confirm` が使える。将来は専用モーダルに差し替え可能 |
| 読み込み中 | 通信中は「読み込み中…」など短い表示があると親切 |

## 12. 今回作らない画面・機能

以下は**画面仕様の対象外**とする。別画面・別機能として将来追加する。

| 対象外 | 理由 |
|---|---|
| ログイン画面 | ログイン機能は今回作らない |
| ユーザー登録画面 | 同上 |
| カテゴリのマスタ管理UI | 選択肢の追加・編集・削除は今回作らない（SQL で投入する） |
| 検索UI | タイトル・本文の全文検索は今回作らない |
| AI要約画面 | AI要約機能は今回作らない |
| 画像アップロードUI | 対象外 |
| 設定画面 | 対象外 |
| 複数ページ構成 | `index.html` 1ファイルに集約 |

### 12.1 今回作らないもの（全体）

- ログイン機能
- ユーザー登録機能
- `user_id` によるデータ分離
- RLS（Row Level Security：行単位のアクセス制御）
- カテゴリマスタの管理UI（追加・編集・削除）
- タグ機能
- 全文検索・複合条件フィルタ
- AI要約機能
- 画像アップロード
- Next.js 化
- React 化
- 外部API連携
- 本番公開手順の整備
- `.mdc` ルールファイル
